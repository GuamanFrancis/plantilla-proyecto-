Tú (Frontend): 
Perfecto. ¿Cómo es el flujo de autenticación exactamente? ¿Qué me devuelve tu API?

Alex (Backend): 
El flujo es así:
1. Haces un POST a `/api/auth/login` con `{ email, clave }`. (Usa demo@demo.com / Demo1234!).
2. Si está todo bien, te devuelvo un JSON: `{ "token": "eyJhb...", "user": { "id": 1, "nombre": "Demo", ... } }`.
3. Tú guardas ese `token` en el `localStorage` (o `sessionStorage`).
4. A partir de ahí, en cada petición a `/api/pacientes`, `/api/citas`, etc., debes enviar el header: 
   `Authorization: Bearer <TU_TOKEN>`.
5. Si el token expira o es inválido, mi API te devolverá un error HTTP 401. Ahí tu frontend 
   debería atrapar ese 401, borrar el token del localStorage y redirigir al usuario a la pantalla de Login.

Tú (Frontend): 
Genial. ¿Cómo me recomiendas estructurar las carpetas en React para no hacer un desastre?

Alex (Backend): 
Te sugiero una arquitectura basada en "Features" (características). Algo así:

src/
 ├── assets/
 ├── components/       # Componentes UI genéricos (Botones, Modales, Inputs)
 ├── config/           # Configuración de Axios (axios.ts)
 ├── context/          # AuthContext.tsx (para saber si el usuario está logueado)
 ├── features/         # Aquí va lo pesado
 │    ├── auth/        # Login.tsx, auth.services.ts
 │    ├── pacientes/   # PacientesList.tsx, PacienteForm.tsx, pacientes.services.ts
 │    ├── citas/       # CitasList.tsx, CitaForm.tsx, citas.services.ts
 │    └── especialidades/
 ├── layouts/          # MainLayout.tsx (Sidebar, Navbar)
 ├── routes/           # AppRouter.tsx (React Router DOM)
 └── types/            # index.ts (Las interfaces que te pasé)
